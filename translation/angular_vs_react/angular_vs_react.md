## 1. 简介

&nbsp; &nbsp; &nbsp; &nbsp;在不久前，我们开发团队需要为 Wix 公司的其中一个旗舰产品 —— html wysiwyg 网页编辑器，选定一种框架技术去进行开发。该产品是一个大型的单页面应用。由于里面含有复杂的逻辑、大量与其他内敛框架（iframe）和服务器之间的通信，以及许多的用户体验性操作，因此我们的团队开发人数多达40人。框架的选定在 ReactJS 和 AngularJS 两者间徘徊不定。这是因为，我们对两者都有着一定的开发经验。一方面，AngularJS 的声明式编程思想使得开发变得易于使用。另一方面，ReactJS 却带给了我们开发的简易性。一系列的概念证明促使我们形成了这次的决胜局。下面，该文将带大家探讨一下两者之间的对决。
